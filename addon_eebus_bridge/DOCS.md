# EEBUS Bridge

This Home Assistant add-on packages the Go bridge from this repository so the
custom `EEBUS` integration can talk to Vaillant EEBUS gateways locally.

## What it does

- Runs the Go `eebus-bridge` service
- Exposes gRPC on port `50051`
- Exposes EEBUS SHIP on port `4712`
- Stores autogenerated certificates in `/data/certs`
- Uses host networking for mDNS/SHIP compatibility

## What it does not do

- Install the Home Assistant custom integration automatically
- Provide full HVAC mode/setpoint control

## Install

1. Add this repository to Home Assistant as an app/add-on repository.
2. Install `EEBUS Bridge`.
3. Start the add-on.
4. Install the `EEBUS` custom integration from this same repository.
5. In the integration config flow, use your Home Assistant host IP and port `50051`.
6. Read the SKI from the add-on logs and trust it in the myVaillant app under
   `Network settings -> EEBUS`.

## Configuration

```yaml
grpc_port: 50051
eebus_port: 4712
vendor: HomeAssistant
brand: eebus-bridge
model: eebus-bridge
serial: ha-001
auto_generate_certificates: true
```

## Notes

- Because the add-on uses host networking, `localhost` is usually not the right
  bridge host from Home Assistant Core. Use the HA host LAN IP instead.
- Certificates persist in `/data/certs`.
- The bridge config is generated on startup from add-on options.
